# Taxpayer Q&A POC (Node + Ollama)

A small, server-rendered proof-of-concept web app for asking natural-language questions about a taxpayer's Form 1040 data. Answers are generated by a locally running [Ollama](https://ollama.com) model, constrained to a structured JSON schema, and grounded strictly in the taxpayer JSON provided — no outside tax knowledge, no guessing.

## How it works

1. Pick a taxpayer from a dropdown (backed by a full-info JSON file).
2. Check any number of predefined questions and/or type free-form questions (one per line).
3. The app sends the taxpayer's JSON data plus the list of questions to a local Ollama model via `/api/chat`, using Ollama's structured-output `format` field to force a `{ answers: [{ questionId, answer }] }` response.
4. Each raw answer is formatted for display based on the question's own known `answerDataType` (currency, date, Yes/No badge, or markdown-rendered text) — the app owns display formatting, not the model.
5. Results are rendered back on the same page alongside the questions asked.

There is no conversation history — every submission is an independent, one-shot request. Nothing is persisted to disk or a database.

## Tech stack

- **Express** — server and routing
- **EJS** — server-rendered views, no client framework/SPA
- **marked** — renders markdown inside `text`-type answers as HTML
- Plain `fetch` to Ollama's REST API (no client library)

## Project structure

```
questionnaire-ai-bot-poc/
  server.js                        # Express app + routes (GET /, POST /ask)
  config.js                        # taxpayer list + Ollama connection settings
  questionnaire.json                # predefined question bank ({ id, text, answerDataType })
  *.taxpayer-info.json              # per-taxpayer 1040 data files (see below)
  lib/
    ollama.js                       # fetch wrapper for Ollama /api/chat (structured output)
    systemPrompt.js                  # fixed system prompt + JSON schema for the model's response
    format.js                       # formatAnswer(dataType, rawValue) -> display HTML
  views/
    index.ejs                       # single page: form + results table
  public/
    style.css                       # minimal styling
```

## Question bank (`questionnaire.json`)

An array of `{ id, text, answerDataType }` objects. `answerDataType` is one of `text | number | money | date | boolean` and drives both the hint given to the model and how the app formats the displayed answer. Questions are phrased in second person ("What is your filing status?"), as if asking the taxpayer directly. Current categories:

- **Personal info**: first/last name, occupation, gender, date of birth, marital status, street address, city, state, zip code
- **Return summary**: filing status, total income, AGI, taxable income, total tax, refund vs. owe, dependents, itemized vs. standard deduction

## Taxpayer data files

Each taxpayer is a `*.taxpayer-info.json` file containing a full 1040 extract (personal info, spouse, dependents, preparer, income/deduction/tax lines, and schedules A/B/D/1/2/3). The taxpayer dropdown is **hardcoded** in `config.js` (id → label → filename), not auto-scanned from disk.

Three demo taxpayers are included, deliberately varied to exercise different combinations of the questionnaire's answer types:

| | Ben | Maria Chen | David Okafor |
|---|---|---|---|
| Filing status | Single | Married Filing Jointly | Head of Household |
| Dependents | None | 2 (with spouse) | 1 |
| Deduction | Standard | Itemized ($34,000) | Standard |
| Refund/Owe | Owes $2,000 | Refund $6,448 | Owes $1,007 |
| Location | Atlanta, GA | San Francisco, CA | Austin, TX |
| Preparer | Yes | Self-filed (none) | Yes |

All taxpayer data is fictional, for demo/testing purposes only.

## Setup

1. Install dependencies:
   ```
   npm install
   ```
2. Make sure [Ollama](https://ollama.com) is running locally and the configured model is pulled (see `config.js` for the model name and base URL, e.g. `gemma3:12b`):
   ```
   ollama serve
   ollama pull gemma3:12b
   ```
3. Start the app:
   ```
   npm start
   ```
4. Open `http://localhost:3000`.

## Configuration

`config.js` holds:
- `taxpayers` — the list of selectable taxpayers (id, label, data file)
- `ollama` — `baseUrl`, `model`, and `numCtx` (context window sized to fit the taxpayer JSON + prompt)

## Design notes

- The model always returns plain-string raw values (digits only for money/number, `YYYY-MM-DD` for dates, `"true"`/`"false"` for booleans) — the app is responsible for all display formatting via `lib/format.js`.
- If a fact isn't present in the taxpayer's data, the model is instructed to answer exactly `"Not available in provided data"` rather than guess.
- This is a single-pass POC (no separate draft-then-verify call) — good enough to demonstrate the approach, not tuned for production accuracy.
